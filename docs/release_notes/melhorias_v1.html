Sugestões de Melhorias — Squad Agentes

Gerado em 11 de maio de 2026  |  Projeto: Contact Management App (FastAPI + Next.js 14)
v1

Fase 1 — Correções imediatas

Crítico ~4h estimadas
  • 1.1 Integrar Navbar no layout
    Navbar.tsx existe e está pronto, mas frontend/src/app/layout.tsx não a inclui (TODO TASK-07 não concluído). Sem ela, o usuário não tem como navegar ou fazer logout pela UI. Solução: descomentar <Navbar /> em layout.tsx.
    Esforço: < 30 min
  • 1.2 Paginação na lista de contatos
    Backend limita a 200 registros sem offset/limit; frontend carrega tudo de uma vez.
    Backend: adicionar skip: int = 0, limit: int = 20 em GET /contatos/, retornar total no response.
    Frontend: controles "Anterior / Próxima" em ContatoTable.
    Esforço: ~3h
  • 1.3 Toast diferenciado para erro vs sucesso
    Mensagem de erro aparece com o mesmo estilo visual de sucesso (verde). Adicionar tipo 'sucesso' | 'erro' ao estado de toast em frontend/src/app/contatos/page.tsx.
    Esforço: ~1h

Fase 2 — Funcionalidades de UX

Alto ~15h estimadas
  • 2.1 Ordenação de colunas na tabela
    Clicar no header da coluna (Nome, Email, Empresa, Data) ordena ASC/DESC. Backend: parâmetros sort_by e sort_order em GET /contatos/. Frontend: ícone de seta nos headers com estado de ordenação.
    Esforço: ~4h
  • 2.2 CRUD completo de usuários
    Só existe POST /usuarios/ (criar). Faltam: GET /usuarios/, GET /usuarios/{id}, PUT /usuarios/{id}, DELETE /usuarios/{id}, PATCH /usuarios/{id}/role (apenas adm).
    Esforço: ~4h (backend) + ~3h (frontend)
  • 2.3 Máscara e validação de telefone
    Campo telefone aceita qualquer string. Adicionar máscara (99) 99999-9999 e rejeitar formatos inválidos. Opções: react-input-mask ou implementação manual.
    Esforço: ~2h
  • 2.4 Ícones nos botões e ações
    Substituir texto puro ("Editar", "Excluir", "Novo") por ícone + texto com lucide-react. Melhora escanabilidade e aparência profissional.
    Esforço: ~2h
  • 2.5 Mensagem de "sem resultados" contextual
    Atualmente: "Nenhum contato encontrado" para ambos os casos. Separar: "Nenhum resultado para '[termo]'" vs "Nenhum contato cadastrado ainda."
    Esforço: < 1h
  • 2.6 Alerta ao sair de formulário sem salvar
    beforeunload + modal de confirmação ao tentar navegar com formulário com alterações não salvas.
    Esforço: ~2h

Fase 3 — Qualidade e segurança

Médio ~13h estimadas
  • 3.1 Cobertura de testes ≥ 80%
    Fixtures em backend/tests/conftest.py estão prontas. Implementar os 22 testes planejados em test_auth.py, test_usuarios.py, test_contatos.py e test_services.py.
    Esforço: ~4h
  • 3.2 PATCH (atualização parcial) em contatos
    Endpoint atual PUT /contatos/{id} exige todos os campos. Adicionar PATCH /contatos/{id} com campos opcionais via Pydantic Optional fields.
    Esforço: ~2h
  • 3.3 Soft delete de contatos
    Adicionar campo deletado_em: datetime | None no modelo Contato. DELETE /contatos/{id} passa a setar deletado_em em vez de apagar fisicamente. Endpoint GET /contatos/lixeira para admins.
    Esforço: ~3h + migração Alembic
  • 3.4 Auditoria básica (quem criou/modificou)
    Adicionar criado_por_id e atualizado_por_id (FK → Usuario) na tabela Contato. Popular automaticamente via dependency require_adm.
    Esforço: ~3h + migração
  • 3.5 Rate limiting na API
    Usar slowapi (FastAPI-compatible) para limitar POST /auth/login (ex: 5 req/min por IP) e prevenir ataques de força bruta.
    Esforço: ~1h
  • 3.6 Validação robusta no frontend com Zod
    Substituir regex manuais por schemas Zod em todos os formulários (login, cadastro, contato). Melhora mensagens de erro e type-safety.
    Esforço: ~3h

Fase 4 — Novas funcionalidades

Valor de negócio ~26h estimadas
  • 4.1 Filtros avançados de contatos
    Filtrar por: empresa específica, data de criação (range), sem email, sem telefone. Painel "Filtros" colapsável na página de lista.
    Esforço: ~5h
  • 4.2 Exportar contatos (CSV / Excel)
    Botão "Exportar" gera arquivo com os contatos da listagem atual (respeitando filtros/busca). Backend: endpoint GET /contatos/export?format=csv.
    Esforço: ~4h
  • 4.3 Importar contatos via CSV
    Upload de arquivo CSV com mapeamento de colunas e preview antes de confirmar a importação.
    Esforço: ~6h
  • 4.4 Dashboard / métricas
    Tela /dashboard com contadores: total de contatos, adicionados este mês, por empresa. Backend: GET /stats com agregações SQL.
    Esforço: ~5h
  • 4.5 Recuperação de senha por e-mail
    POST /auth/forgot-password envia link de reset; POST /auth/reset-password valida token e redefine senha. Requer configuração de SMTP (ex: SendGrid, Resend).
    Esforço: ~6h
  • 4.6 Tags / categorias em contatos
    Campo tags: list[str] no modelo Contato (JSON ou tabela separada). Filtrar por tag na listagem.
    Esforço: ~5h

Fase 5 — Infraestrutura e operação

Operacional ~16h estimadas
  • 5.1 Docker Compose
    docker-compose.yml com serviços: api (FastAPI), frontend (Next.js), db (PostgreSQL).
    Esforço: ~3h
  • 5.2 Migrar de SQLite para PostgreSQL
    Atualizar DATABASE_URL para postgresql+asyncpg://..., ajustar tipos (UUID, arrays). Rodar testes na nova engine.
    Esforço: ~3h + testes
  • 5.3 CI/CD com GitHub Actions
    Pipeline: lint → testes backend → build frontend → deploy (Railway, Render ou Vercel).
    Esforço: ~4h
  • 5.4 Monitoramento de erros (Sentry)
    Integrar Sentry no backend (FastAPI middleware) e frontend (@sentry/nextjs).
    Esforço: ~2h
  • 5.5 Dark mode
    Tailwind dark: classes + toggle no Navbar + persistência via localStorage.
    Esforço: ~4h
